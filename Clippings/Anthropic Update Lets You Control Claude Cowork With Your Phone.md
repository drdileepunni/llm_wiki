[Anthropic this week](https://x.com/felixrieseberg/status/2034005731457044577) introduced an update called Claude Dispatch that lets people control Cowork from a phone through what the company calls a persistent thread across desktop and mobile. Announced as a “research preview," as a new feature in Claude Cowork, Dispatch lets users kick off and monitor desktop tasks from a phone, pushing Claude closer to an operating layer for daily work rather than just a chat window.

The work still runs on the desktop computer and so local files stay there. The phone does not actually run or coordinate the activities, but it becomes the place where work gets assigned, checked and picked up again later. For most business users, AI still lives inside a familiar box. You open a browser, desktop app or terminal client, ask a question or perform a task and move on. Anthropic’s Cowork product has already been pushing beyond that model by giving Claude access to local files, browser tools and connected systems on a user’s own machine.

The mobile extension takes the next logical step. A user can start a task from a phone, let the desktop machine handle the file access and execution, then return to the result without sitting in front of the computer the whole time.

Anthropic says the feature is rolling out to Pro and Max subscribers, with Max users getting it first and Pro following shortly after. The company’s support materials describe the experience as one ongoing thread with the same context available across devices. That means users are not starting over every time they switch screens. The conversation persists and so do the tasks.

The agent market is already nudging users in the direction of mobile and messenger-first agentic AI interaction. OpenClaw, an open-source assistant built to work through WhatsApp, Telegram, Slack, Signal and a long list of other chat channels, treats messaging as the primary interface rather than a side feature.

That matters because it reflects a wider change in user expectations. People increasingly want AI to behave less like software they visit and more like a capable operator they can text. Anthropic’s Cowork feature originates with Claude’s more web, desktop and terminal focus, but this newest addition shows that Anthropic is paying attention to the rapidly evolving agent market. The center of gravity is shifting toward messenger-first, mobile-friendly control.

In this case, Dispatch is not an attempt at an OpenClaw competitor. The existing locations for Claude operation remain, the files stay where they live and the configured tools sit where Claude can perform the tasks inside Cowork. The phone just becomes a lightweight command surface.

Anthropic’s documentation says users can assign tasks from the mobile app, let Claude continue working on the desktop and receive a finished output later. The examples are practical, including summarizing recent work, pulling together a brief, comparing materials across folders or producing a report while the user is away from the desk.

The goal is to remove small bits of drag that pile up over the day. The gap between “I need this done” and “I am at the machine that can do it” has always been one of those irritants. Workers compensate for it with delay, with workaround or with forgotten tasks. Anthropic’s update chips away at that gap.

Tools like Claude are no longer just a place to ask for text output or analysis. It is becoming a layer that can act inside an existing desktop environment, then report back through a mobile one. That makes the phone less of a productivity tool in its own right and more of a dispatch console for the tools that already exist elsewhere.

Rather than running and coordinating everything in cloud-based environments, Anthropic is betting that a local-first setup will appeal to users who want more capability at their desktop or in their networks without sending every sensitive document or workflow into a new external environment. For a bank, hospital, law firm or industrial company, the question is rarely whether AI can generate output. The real question is where the work runs, what it can touch and how much of the environment must be opened up to make it useful.

Anthropic makes it clear that phone-based control does not change that underlying model. That is an important design choice for companies dealing with internal documents, regulated data, proprietary research or ordinary corporate caution. While it does not eliminate risk, it narrows one source of concern by leaving execution anchored to the machine where the work already resides. This local-first posture may end up being one of the more commercially relevant aspects of the release.

Anthropic deserves some credit for being direct about the downsides of Cowork and Dispatch. Its support article warns that the feature can trigger actions on the user’s computer from the phone. Those actions may include reading, moving or deleting files, controlling the browser or interacting with tools and services that Cowork already has permission to use.

This is the trade that comes with every step from chatbot toward agent. The more useful the system becomes, the more consequential its access becomes. A model that can draft text is one thing. A model that can manipulate files, use authenticated tools and continue working when the user is not physically present is another.

Anthropic places limits around the experience. The computer must be awake. The Claude desktop app must remain open. Dispatch is a feature that is still labeled a research preview. Still, enterprise buyers will read this release through a security lens before they read it as a productivity story. Mobile convenience has a long history of expanding risk surfaces. The same pattern showed up in mobile banking, remote admin tools and cloud collaboration software. Claude Cowork may face the same test. The feature does not need to win over every company at once. It only needs to prove that the operational upside is strong enough for a defined class of users.
